McAfee全球威胁情报的AdvancedThreat Research团队和McAfee Labs恶意软件运营部门发现了针对核，国防，能源和金融公司的新全球运动。 该活动，Sharpshooter利用内存植入下载和执行第二阶段植入的木马程序 - 我们称之为Rising Sun。 根据我们的分析，Rising Sun植入的木马程序使用来自Lazarus Group 2015年后门Trojan Duuzer的源代码，在新的框架中渗透这些关键行业。

sharpshooter的攻击手法与Lazarus组织的众多技术比较相似，虽然不能立即得出结论或许可能存在虚假旗帜。 我们的研究重点是该攻击组织如何运作，对全球影响以及如何检测攻击。

该攻击活动虽然伪装成合法的招聘活动，对进一步攻击做监控收集信息。 我们的分析还表明与其他招聘活动相关的类似技术。
这项研究使用2015年后门Duuzer的代码发现了一个新的植入代码，该代码最后一次出现在2015年针对目标是韩国和日本。除了Rising Sun，我们从那时起就没有看到其他变种。

1.全球影响

在2018年10月和11月，Rising Sun恶意软件已经出现在全球87个组织中，主要是在美国，基于McAfee的分析。 基于具有类似行为的其他活动，大多数目标组织都是说英语或有英语的办公室。 该攻击使用recruiting（招募）作为诱饵文件来收集有关目标个人或管理与感兴趣行业相关的数据的组织信息。 迈克菲高级威胁研究团队观察到，大多数目标都是国防和政府相关组织。

2.活动分析

这项行动于10月25日开始。一系列恶意文件中载有姓名为Richard（理查德）的作者。 这些文档包含韩语信息，表明它们是使用韩语版的Microsoft Word创建的。 所有恶意文档都有针对未知公司职位的英文职位描述标题，由美国的IP地址和Dropbox服务分发。 这些文档包含一个恶意宏，利用嵌入式shellcode将Sharpshooter下载器注入到Word的内存中。 一旦Word进程被感染，下载器就会执行第二阶段运行Rising Sun.
下载程序的shellcode大小为3.1KB，获取另一个植入程序地址在hxxps：//www.kingkoil.com.sg/query.php。

由winword.exe中的Visual Basic for Applications宏执行的shellcode充当第二阶段植入的简单下载程序。 shellcode使用第二阶段有效负载感染端点需要四个步骤：

  1. 它通过使用硬编码字节填充字符串数组来构建库和API名称。 （字符串构造一次完成1个字节。）此技术用于构造shellcode中的所有字符串，包括控制服务器信息。

  2. 它使用LoadLibraryA（），GetProcAddress（）解析库和API：

    • shfolder.dll

  • ntdll.dll

    • kernel32.dll

    • shell32

  • LoadLibraryA

    • GetProcAddress

    • URLDownloadToFileA

    • SHGetFolderPathA

    • strcpy

    • strcat

    • CreateProcessA

    • memset

    • ShellExecuteA

  1. 植入木马程序从其控制服务器下载两个文件：

Advanced Threat Research团队发现了同一作者的另一份PDF文档（10mins.PDF）。 它似乎是一个与智能手机相关的调查问卷。 此文档与两个相关的恶意文档托管在同一服务器上。 调查问卷来自于一家专门从事反欺诈保护和财务合规的大数据分析公司。

4、Rising Sun 行为分析

Rising Sun木马程序是一个功能齐全的模块化后门，可对受害者的网络进行侦察。

此种植体首先通过动态API解析构建并导入：LoadLibrary（）/ GetProcAddress（）。 库和API名称在木马程序中硬编码为DWORD / WORD值，并包含大小为0x147字节的字节。 使用具有密钥0xC8的简单单字节XOR方案来解密该blob数据。
用于构建库和API名称的这种方案是Lazarus恶意程序经常使用的字节块字符串构造技术的变体。 该计划通常涉及如下内容：

  • 恶意程序中DWORD / WORD /字节块形式的硬编码库和API名称。

  • 在恶意程序执行期间为这些硬编码值分配变量。

  • 构造由要解析的库和API名称组成的字符数组。

  • 这些阵列可能必须使用像单字节XOR解码方案那样简单的东西来解码。

  • 使用LoadLibrary（）/ GetProcAddress（）现在使用构造的名称数组解析库和API。

恶意程序的配置数据使用RC4算法加密。 恶意程序在运行时解密配置数据并与控制服务器通信。 从恶意程序解密的地址信息如下：

  • hxxtp://34.214.99.20/view_style.php

  • hxxp://137.74.41.56/board.php

  • hxxps://www.kingkoil.com.sg/board.php

恶意程序从被控端主机获取以下数据并将其传送到c&c控制服务器包括如下内容：

  • 网络适配器信息

  • 计算机名称

  • 用户名

  • IP地址信息

  • 本机系统信息

  • 来自注册表的OS产品名称：SOFTWARE\MICROSOFT\Windows NT\CurrentVersion | ProductName

5、sharpshooter与Lazarus的同源性分析

将攻击归咎于任何威胁组织往往充满挑战，包括其他威胁行为者可能的“虚假旗帜”行动。 仅凭技术证据不足以证明组织之间存在的关联性。 然而，根据我们的分析，这一行动与其他Lazarus组织的攻击有着惊人的相似之处; 因此我们提出这一观点，以便于进一步分析。 虽然这些相似之处指向Lazarus，但我们也必须考虑存在假旗的可能性。

  • 恶意Word文档是在韩语环境中创建的。 （代码页是韩文。）

  • 恶意程序使用了我们在多个Lazarus种植体中观察到的动态API技术的变体。

  • 该操作非常类似于2017年针对美国国防和能源部门的Lazarus业务。 不论从技术，策略和程序与之前的操作相非常相似。

  • Rising Sun是Lazarus后门Duuzer的演变，它于2015年流传，目标是韩国。

Rising Sun与Duuzer有部分共享代码，该特征由安全社区确定属于Lazarus。 我们比较了以下样本并详细说明了它们的相同点和不同点。
用于比较的信息：

Rising Sun: f3bd9e1c01f2145eb475a98c87f94a25
Duuzer: 73471f41319468ab207b8d5b33b0b4be

尽管Rising Sun和Duuzer使用的解密方案不同，但两种恶意程序都使用类似的配置信息来驱动其侦察能力：

两种木马都使用相同的技术来构建和解码库和API以进行动态API解析。 我们在前面的部分中解释了这种技术（a variant of byte-chunk library/API name construction）。 尽管由Duuzer中的库/ API字符串组成的编码数据blob的大小为0x181字节，并且使用0x30作为XOR密钥进行解码，但是Rising Sun中的编码数据blob的大小为0x147字节，并使用0xC8作为XOR密钥进行解码。
两个木马之间的另一个相似之处是一些解码的库名称由随机字符组成。
例如，Duuzer将以下库名的随机字符大写：

uSEr32.dlL
Rising Sun在这些库名称中有相似的名称：
vErsIon.dll
advapI32.dLL

恶意程序的相似性是使用多种功能以及代码结构和API中直接匹配，以实现这些功能。
以下功能是直接匹配：
侦察（收集初步系统信息）

两种恶意程序 在初始侦察期间都从被控主机中捕获相同的信息。 信息的顺序和API /代码签名完全匹配。
两种恶意程序窃取的信息：
网络适配信息
计算机名称
用户名
IP地址信息
本机操作系统信息
来自注册表的os产品名称：SOFTWARE\MICROSOFT\Windows NT\CurrentVersion | ProductName

5.1 Rising Sun 和Duuzer的相同点

1.两者执行命令相似

两个恶意程序都使用cmd.exe执行命令，并将输出重定向到临时文件：

cmd.exe /c “> <%temp%>\.tmp” 2>&1
两种恶意程序都支持本地更改目录，而无需通过shell执行cd命令。
支持的cd命令：
cd
cd.
cd\

2.获取驱动信息相似

两种恶意程序都使用相似的代码签名收集相同的数据：

Drive type
Total number of bytes on disk
Total number of free bytes on disk
Name of a specified volume

3.启动进程相似

两种恶意程序都使用相同的API和标志在主机上启动新进程。

4.获取进程相似

两种恶意程序都会获取完全相同的进程信息：

Process name
Process creation time
Process exit time
Process kernel mode time
Process user mode time

5.终止进程相似

两种恶意程序都支持基于以下任一方式终止在系统上运行的进程的能力：

Process Name
Process ID

6.获取文件的时间

两个恶意程序实现相同的功能

Find files based on a filename search string (for
example, . or *.txt)
For each file found, get the following times:

  • File creation time

  • Last access time (including read, write, or execute
    operations)

7.读文件

两个恶意程序都可以读取控制服务器指定的文件内容并执行文件的内容。

8.清除进程信息

两个恶意程序中没有相似性

9.写文件

两种恶意程序都可以使用相同的操作序列将C&C主控端提供的内容写入磁盘（文件路径也由c&c控制服务器指定）：

从控制服务器获取文件路径并创建与文件路径对应的文件。
使用特定恶意程序的通信机制从控制服务器获取要写入文件的内容。
将内容写入文件路径后，向控制服务器发送成功或失败响应。

10.删除文件

如果C&C控制服务器指定的文件不是目录，则两个木马都可以删除控制服务器指定的文件。

11获取其他信息

两种恶意程序都具有相同的功能来获取指定目录中文件的文件信息，
包括以下数据：

File attributes
File size
File creation time
Last access time
File write time
MZ compile time

12.连接相同的C&C地址

两个种植体都使用相同的操作，API和代码签名与指定IP地址连接：

  • 通过指定的端口号测试与指定网络IP地址的连接。

  • 仅尝试连接到网络地址。

  • 根据连接尝试，向控制服务器发送成功或失败响应。

13.两个恶意程序都可以修改文件属性

文件属性（隐藏，系统等）
如果文件是MZ，则文件的编译时间戳也会在PE头中修改。

14.更改文件路径

两种恶意程序都可以更改文件属性并将文件移动到其他位置。

5.2 Rising Sun 和Duuzer的不同点

两个家族在某些方面存在一些差异性。

1.通信机制：

Duuzer使用简单的基于套接字的通信机制从其控制服务器发送和接收数据。
Rising Sun使用基于HTTP的机制。 这种差异可能是攻击版本的增强，因为基于HTTP的机制能够更好的绕过网络入侵防御系统的检测。

2.编码方式不一致

编码不一致:除了库和API名称构造和解码之外，恶意程序中使用的加密方式是完全不同的。 虽然Duuzer使用自定义的XOR来解码其配置数据，但Rising Sun使用RC4算法解密。

6.总结

我们对新型木马的发现是针对性攻击试图获取智能的另一个例子。 恶意软件分几步进行。
初始攻击向量是一个包含武器化宏的office文档，用于下载真正的后门程序，该阶段在内存中运行并收集情报。 受害者主机的数据被发送到C&C控制服务器以被攻击者监控，然后攻击者确定接下来的步骤。

我们以前没有观察到这种恶意程序。 根据我们的分析，我们发现世界各地不同行业的多名受害者都发现了这些特征。 Sharpshooter与Lazarus Group恶意软件的相似之处令人震惊，但这并不能确保两个组织的同源性。 这次攻击只是第一阶段的侦察行动，还是会有更多的活动信息吗？ 我们将继续监控此活动，并在我们收到更多信息样本后进一步报告。
